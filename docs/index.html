pofile-ts implements the GNU gettext PO file format and is optimized for seamless integration with translation management platforms.

GNU gettext Crowdin

Crowdin-optimized output: By default, pofile-ts uses a compact multiline format that avoids unnecessary diffs when syncing with Crowdin. See lingui#2235 for background.

Quick Start

Get up and running in seconds with pofile-ts:

import { parsePo, stringifyPo, createItem } from "pofile-ts"

// Parse a PO file
const po = parsePo(`
msgid "Hello"
msgstr "Hallo"
`)

console.log(po.items[0].msgid)   // "Hello"
console.log(po.items[0].msgstr)  // ["Hallo"]

Create a new PO file

import { createPoFile, createItem, stringifyPo } from "pofile-ts"

const po = createPoFile()
po.headers["Content-Type"] = "text/plain; charset=UTF-8"

const item = createItem()
item.msgid = "Welcome"
item.msgstr = ["Willkommen"]
po.items.push(item)

console.log(stringifyPo(po))

API Reference

Functions

All exports are named functions ‚Äî no classes, no default export.

Function Description
parsePo(content) Parse a PO file string into a PoFile object
stringifyPo(po, options?) Serialize a PoFile to string
createPoFile() Create a new empty PoFile
createItem(options?) Create a new translation item
stringifyItem(item, options?) Serialize a single item to string

File I/O (Node.js)

Use native fs module for file operations:

import { readFile, writeFile } from "node:fs/promises"
import { parsePo, stringifyPo } from "pofile-ts"

// Load from file
const content = await readFile("messages.po", "utf-8")
const po = parsePo(content)

// Save to file
await writeFile("output.po", stringifyPo(po))

Serialization Options

Control the output format when serializing PO files. The defaults are optimized for compatibility with translation platforms like Crowdin. 

import { stringifyPo } from "pofile-ts"

// Default: compact format, 80 char line width
stringifyPo(po)

// Traditional GNU gettext format (empty first line for multiline)
stringifyPo(po, { compactMultiline: false })

// Disable line folding
stringifyPo(po, { foldLength: 0 })
Option Default Description
foldLength 80 Max line width before folding. Set to 0 to disable.
compactMultiline true Use compact format for multiline strings (Crowdin-compatible)

Multiline Format Comparison

Strings containing newlines (\n) can be formatted in two ways. Both are valid PO syntax and represent the exact same msgid value: 

// Compact (default) ‚Äî Crowdin-compatible
msgid "First line\n"
"Second line"

// Traditional GNU gettext ‚Äî with empty first line
msgid ""
"First line\n"
"Second line"

// Both parse to the same value:
// msgid = "First line\nSecond line"

Why does this matter? The difference is purely cosmetic ‚Äî no translation data changes. But without consistent formatting, you'll see constant diffs in Git when syncing with Crowdin. The compact format avoids this by matching Crowdin's normalization.

Working with Plurals

PO files support plural forms. The number of plural forms depends on the language.

import { createItem } from "pofile-ts"

const item = createItem()
item.msgid = "One item"
item.msgid_plural = "{count} items"
item.msgstr = [
  "Ein Element",      // singular
  "{count} Elemente"  // plural
]

po.items.push(item)

Migrating from pofile

This package is a modernized fork of the original pofile package with a functional API.

Requirements

  • Node.js 20+ or modern browsers
  • ESM-first with full TypeScript support
  • Named exports only ‚Äî no default export

API Changes

The class-based API has been replaced with pure functions:

// ‚ùå Old API (pofile 1.x)
import PO from "pofile"
const po = PO.parse(content)
const item = new PO.Item()
po.toString()

// ‚úÖ New API (pofile-ts)
import { parsePo, stringifyPo, createPoFile, createItem } from "pofile-ts"
const po = parsePo(content)
const item = createItem()
stringifyPo(po)

The load() and save() methods have been removed. Use native file system APIs instead: 

// ‚ùå Old API (pofile 1.x)
PO.load("messages.po", (err, po) => {
  if (err) throw err
  // use po
})

// ‚úÖ New API (pofile-ts)
import { readFile } from "node:fs/promises"
const content = await readFile("messages.po", "utf-8")
const po = parsePo(content)
