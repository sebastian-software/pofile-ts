**pofile-ts** is a robust TypeScript library for reading and writing GNU gettext PO files. Zero dependencies, full type safety, runs everywhere.

## Features

- **Parse PO Files** — Read PO files from strings, 8× faster than alternatives
- **Serialize PO Files** — Convert PO data back to strings, 5× faster than alternatives
- **Full PO Support** — Headers, comments, flags, plurals, and message context
- **CLDR 48 Plural Data** — 100% compliant for all major languages
- **ICU Conversion** — Convert between Gettext and ICU MessageFormat
- **Zero Dependencies** — No Node.js APIs required, works in browsers too
- **Tree-Shakeable** — Only bundle what you use (~11KB full, ~5KB core)
- **TypeScript First** — Full type definitions for excellent IDE support

## Installation

```bash
npm install pofile-ts
```

## Quick Example

```typescript
import { parsePo, stringifyPo } from "pofile-ts"

// Parse a PO file
const po = parsePo(`
msgid "Hello"
msgstr "Hallo"
`)

console.log(po.items[0].msgid) // "Hello"
console.log(po.items[0].msgstr) // ["Hallo"]

// Serialize back to string
console.log(stringifyPo(po))
```

## Performance

Benchmarked with 10,000 entries (~10% plurals) on Apple M1 Ultra, Node.js 22:

| Library        |       Parsing | Serialization |
| -------------- | ------------: | ------------: |
| **pofile-ts**  | **185 ops/s** | **217 ops/s** |
| gettext-parser |      27 ops/s |      53 ops/s |
| pofile         |       8 ops/s |     100 ops/s |

_Relative performance (×faster) is consistent across different hardware._

## Bundle Size

The library is fully **tree-shakeable**. Import only what you need:

| Import                         | Gzipped |
| ------------------------------ | ------: |
| Full library                   |   ~11KB |
| `parsePo` + `stringifyPo` only |    ~5KB |
| Add CLDR plural helpers        |    +3KB |
| Add ICU conversion             |    +2KB |

All exports are named exports — modern bundlers automatically eliminate unused code.
